Introduction to Raster Data: Gridded World Population

Preamble

The field of human population mapping is constantly evolving, leveraging the increasing availability of high-resolution satellite imagery and the advancements in the field of machine learning. In recent years, the emergence of global built-area datasets that accurately describe the extent, location, and characteristics of human settlements has facilitated the production of new population grids, with improved quality, accuracy, and spatial resolution.

In this demonstration lab, we will utilize the GPW v4 Population Count Dataset hosted by Columbia University. This dataset features a estimated population value (count) per grid cell of approximately 5km spatial resolution (2.5 minute). The following short video introduces the data and its production methodology:

We will explore raster symbolization and utilize the Raster Calculator to produce a difference value between 2010 and 2020 across the GPW dataset. Once we achieve our raster result, we will then classify the raster values into an ordinal scale of population decline/growth between the years 2010 and 2020.

Data

  • Demonstration Lab Data

  • There are two raster datasets in the data directory. They originate from the Gridded Population of the World V4 (GPW) based on two temporal variables - 2010 and 2020 population counts (number of persons per pixel).

Step 1:

Open ArcGIS and point ArcCatalog to the c5.lab.data.gpw-v4-pop_count_2010_2020 lab data. Remember to uncompress (unzip) the data directory before proceeding. Unlike previous week’s Vector and Delimited Text file types, we will be loading Rasters this week.

Import both 2010 and 2020 Gridded Population Rasters into TOC:

Activate the Pixel Inspector tool from main menu > customize mode > commands > Pixel Inspector:

With both rasters loaded, install the tool and enable it into a main tool bar (it will install into any of the main toolbars). Find cell areas at/near a large urban area and move the tool over this geography:

In this demonstration lab, this difference between 2010 and 2020 across both raster extents will be derived and mapped. Prior to calculating difference, symbolization will be explored.

Step 2:

Within the 2010 layer, navigate to Layer Properties > Symbology. Choose Stretched. Point the Stretch Type setting to the Percent Clip option. A standard data range is set from 2% to 98% of the data values, meaning that the outliers will not be used to set the minimum and maximum values, resulting in a much more representative visualization.

Next, run the same symbolization for the 2020 layer, then check results:

If desired, raster values can also be symbolized just like the standard vector classification methods covered in Class 4. Point to Classified type, and apply a Natural Breaks 5 class method to the 2010 layer:

Step 3:

Make sure both 2010 and 2020 population rasters are still loaded; the symbolization of the raster layers does not matter at this step:

Save the .mxd project before moving to calculations. Open the Raster Calculator via Search. If you receive an install warning, simply enact the Spatial Analyst Extension from Customize > Extensions:

Within the Raster Calculator, raster bands are displayed in upper left of the tool interface. The calculation that will be utilized to derive change is simple raster math - in this case subtraction formatted as follows:

\[ Population Raster2020 - Population Raster2010 \]

"gpw_v4_population_count_rev11_2020_2pt5_min.tif" - "gpw_v4_population_count_rev11_2010_2pt5_min.tif"

Output the result to the Default.gdb as population_change_2010_2020.tif:

Step 4:

  • Visualize the results with a Manual classification using 4 class breaks. Here we give an ordinal value to the colors beyond their original position within the dataset. Four discrete values with corresponding unique colors will be used to break the broad theme of population change into meaningful categories:

    • Decline | -100
    • Neutral | 100
    • Growth | 1,000
    • High Growth | 2,000,000

Within the population_change_2010_2020.tif layer , navigate to layer properties > symbology. Choose Manual method:

Each of the four discrete breaks will be entered manually. Start with Decline, setting the break to -100 and color to dark blue, then selecting Apply. Continue with the other values in the proposed ordinal scale from Decline to High Growth:

Zoom into larger scale geographies (the northeast US shown below) and notice population spatial patterns. ‘Rustbelt’ cities in the northeast US feature prominently in the Decline class:

Zoom further to the City of Detroit and City of Flint to see the large scale pattern 2010 - 2020:

Step 5:

In addition to performing spatial operations between raster layers, a typical analysis tactic is to create statistic aggregations of raster cells within vector features. This is known as a Zonal Statistics.

To start, input ne_10m_admin_0_countries_lakes.shp - country polygons - to the .mxd:

Next, use Search to find Zonal Statistics as Table tool:

Open the tool and populate as follows, setting the statistic to SUM and exporting the tabular result to the Default.gdb:

Once complete, Table Join the resulting zone table to the ne_10m_admin_0_countries_lakes.shp using the join field ADM0_A3:

Warning

For records that do not join, a NULL value will be produced. If selected, it will be clear that the NULL value in this table join is the result of land mass polygons that do not possess input cells, i.e. there is no population recording in these particular geographies.

Next, classify the variable SUM via Natural Breaks with 5 classes. While SUM has not been normalized for a population density value, the raw count for countries is still valid showing which countries possess most/least population counts.

Concluding Remarks

In this fifth demonstration lab, we explored important dimensions of raster data; conducted a simple overlay analysis via the Raster Calculator, and applied an ordinal scale to discrete class breaks to produce a global population result that helps us understand both decline and growth at both small and regional scales. Further, we explored Zonal Statistics - an important, valuable aggregation tool commonly applied to raster layers.

The Gridded Population of the World (GPW), v4 used in this demonstration lab is one of several global population datasets readily available for GIS analysis. Two products worth exploring further are as follows:
